Capabilities

Enterprise-grade PKI, quantum-safe by default

Every component is designed for sovereign deployment — air-gapped capable, multi-tenant, and cryptographically auditable.

🔐

Post-Quantum Ready

Native support for KAZ-SIGN (Malaysia's local PQC algorithm) and NIST ML-DSA, with classical RSA and ECC for migration compatibility.

🏢

Multi-Tenant Architecture

Each tenant gets an isolated database with dedicated schemas. Complete data separation for government and enterprise compliance.

🔑

Cryptographic Credentials

Every user holds dual keypairs — signing and KEM. Private keys are password-encrypted. Public keys are attested by the creating admin.

🏛

Key Ceremony Manager

Multi-phase ceremony with threshold Shamir secret sharing. Requires multiple key managers and auditor sign-off. Full audit trail.

🖥

HSM Integration

PKCS#11 interface supports SoftHSM2 for testing and hardware HSMs (Thales Luna, AWS CloudHSM) for production key protection.

✅

OCSP/CRL Validation

Real-time certificate status via OCSP responder and CRL distribution. ETS-cached for high-throughput validation queries.

How It Works

From zero to issuing certificates in four steps

01

Platform Setup

Create a tenant with an isolated database and four dedicated schemas — CA, RA, validation, and audit.

02

CA Bootstrap

Initialize the first admin with dual keypairs, Keypair ACL, and system keypairs via secure key ceremony.

03

RA Configuration

Set up certificate profiles, validation policies, and API keys for CSR submission and processing.

04

Issue Certificates

Submit CSRs via portal or REST API. Auto-validate, approve, sign with PQC or classical algorithms, and distribute.
